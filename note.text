নেস্টেড অবজেক্টের ক্ষেত্রে ধাপে ধাপে চেইন করে চেক করো।
ডাইনামিক API বা ডেটা ফেচ করলে Optional Chaining অনেক সেফ!






map → প্রত্যেকের জন্য নতুন কিছু দরকার?
forEach → কিছু লুপ করে দেখাতে চাই?
filter → শর্ত মিলে কয়েকটা আইটেম দরকার?
find → শর্ত মিলে প্রথম একটা আইটেম দরকার?









map → যখন প্রতিটি এলিমেন্টে কাজ করে নতুন অ্যারে লাগবে।
forEach → যখন শুধু লুপ চালিয়ে কিছু প্রিন্ট করতে চাও, কিন্তু রিটার্ন দরকার নেই।
filter → যখন শর্ত অনুযায়ী একাধিক আইটেম দরকার।
find → যখন শর্ত অনুযায়ী প্রথম একটা আইটেম দরকার।
